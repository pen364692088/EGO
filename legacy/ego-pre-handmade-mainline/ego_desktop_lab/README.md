# EGO Desktop Lab

`ego_desktop_lab` is the current active innovation and product-lab spine for
the v7 Self-Maintaining Agency Kernel work.

This is a lab authority only. It does not override
`docs/PROGRAM_STATE_UNIFIED.yaml`, does not write the formal evidence ledger,
does not mutate OpenEmotion state, and does not affect EgoCore runtime replies.

## Current Lab Spine

The lab owns deterministic mechanism proof for:

- v7 Self-Maintaining Agency Kernel
- behavior option registration and proposal-only selection views
- bounded affective / drive dynamics as viability pressure, not persona text
- DecisionView and shell observability
- replay determinism and plasticity tests

The old `subject_system_v1_governed_proactivity` runtime/Telegram lane is a
reference and future integration candidate for this lab work. It is not the
current innovation blocker.

## Non-Goals

- No formal mainline promotion in this slice.
- No formal evidence ledger writes.
- No runtime reply mutation.
- No OpenEmotion state mutation.
- No file, tool, desktop, or external-send authority.
- No consciousness, alive, live autonomy, runtime efficacy, or user benefit
  claim.

## Future Integration Rule

Any future connection to EgoCore, OpenEmotion, Telegram, or desktop tooling must
start as a shadow/event tap. The lab may observe copied events and emit shadow
DecisionView records, but it must not write back state, select runtime replies,
or bypass host gates.
