# P0 Known Limitations

## 已知限制

### 1. 任务规划简单
**现状**: `TaskPlanner.create_steps()` 目前只是简单地将任务描述拆分为单步。

**P1 建议**: 
- 接入 LLM 进行智能任务分解
- 支持多步骤复杂任务

---

### 2. 工具支持有限
**现状**: 只实现了基础的 file、shell、python 工具。

**P1 建议**:
- 添加更多工具（web search、API 调用等）
- 工具链编排

---

### 3. 记忆系统简单
**现状**: Memory 模块存在但只做基础存储，没有智能检索和上下文注入。

**P1 建议**:
- 实现记忆检索和注入
- 任务间记忆共享

---

### 4. 恢复机制基础
**现状**: Checkpoint 存在，但 bot 重启后需要手动 `/resume`。

**P1 建议**:
- 自动恢复活动任务
- 断点续传

---

### 5. 自然语言任务识别
**现状**: P0 采用最稳妥策略，不自动识别任务意图。

**P1 建议**:
- 添加自然语言任务意图识别
- 二次确认后创建任务

---

## 未做部分

### 不在 P0 范围
以下功能明确不在 P0 范围内，留待后续：

1. **多 Agent 编排** - 不在范围内
2. **复杂 UI** - 不在范围内
3. **新调度框架** - 不在范围内
4. **自然语言自动建任务** - P1 考虑
