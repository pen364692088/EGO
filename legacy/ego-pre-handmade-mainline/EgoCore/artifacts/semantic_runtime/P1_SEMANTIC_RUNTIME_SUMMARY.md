## 当前问题（P0 之后）

P0 修复了基本的命令注册和误建任务问题，但仍存在：

1. **交互太重**: 用户必须 `/new` → `/run` 才能执行任务
2. **无语义理解**: 所有自然语言都返回帮助文本，不会自动执行
3. **继续绑定缺失**: "继续"类消息无法绑定到活动任务
4. **回复机械**: 所有回复都是模板，没有差异化

---

## 本次引入的语义分流

### 5 类意图分类

| 意图 | 触发条件 | 处理方式 |
|------|----------|----------|
| CHAT | 问候、寒暄 | 友好回复，提供建议 |
| QUESTION | 问句、咨询 | LLM 直接回答 |
| NEW_TASK | 任务请求动词 | 自动创建+执行首步 |
| CONTINUE_TASK | 继续/恢复词 | 绑定活动任务+继续 |
| COMMAND | `/` 开头 | 命令路由 |

### 自动首步执行

- NEW_TASK 意图 → 自动创建任务 → 自动执行第一步
- 安全操作自动执行（读取、分析、查询）
- 高风险操作需要确认

### 继续任务绑定

- "继续"、"接着做"、"还有呢" → 绑定最近未完成任务
- 自动恢复上下文
- 自动执行下一步
