# P1 Intent Routing Spec

## 意图分类定义

### 1. CHAT (聊天)

**定义**: 问候、寒暄、状态确认类消息

**触发条件**:
- 纯问候词：你好、hi、hello、在吗、嗨
- 状态询问：你还在吗、怎么样
- 确认词：好的、ok、收到、谢谢

**进入流程**:
1. 检查是否有活动任务
2. 如有任务，返回任务状态概览
3. 如无任务，返回友好问候和建议

---

### 2. QUESTION (问答)

**定义**: 普通问答、咨询、解释请求

**触发条件**:
- 以"为什么"、"如何"、"怎么"开头
- 包含"是什么意思"、"是什么"
- 以问号结尾
- 包含"解释"、"说明"

**进入流程**:
1. 直接调用 LLM 回答问题
2. 如问题与任务相关，建议用户创建任务
3. 返回简洁答案

---

### 3. NEW_TASK (新任务)

**定义**: 明确的任务执行请求

**触发条件**:
- 以"帮我"、"请帮我"开头
- 以动词开头：读取、检查、分析、生成、创建
- 包含"并总结"、"并分析"
- 包含"什么问题"、"什么核心问题"
- 查询类：查询、获取、列出

**进入流程**:
1. 提取任务内容
2. 检查是否高风险操作
3. 自动创建任务
4. 自动执行第一步
5. 返回阶段性结果

---

### 4. CONTINUE_TASK (继续任务)

**定义**: 继续、恢复、追问当前任务

**触发条件**:
- 继续：继续、接着做、接着、继续执行
- 追问：还有呢、还有什么、接下来
- 任务询问：上个任务、上一个任务、刚才的任务
- 状态：怎么样了、进展如何
- 恢复：恢复、resume

**进入流程**:
1. 查找活动任务（优先级：running > blocked > paused > planning）
2. 查找最近未完成任务
3. 绑定任务后自动执行下一步
4. 返回执行结果

---

### 5. COMMAND (命令)

**定义**: 显式命令输入

**触发条件**:
- 以 `/` 开头
- 如：/new, /run, /status, /help 等

**进入流程**:
1. 解析命令和参数
2. 路由到对应命令处理器
3. 返回命令结果

---

## 歧义处理

### 当分类置信度低时：

1. **短消息 (< 10 字符)**: 默认 CHAT
2. **长消息无明确意图**: 默认 NEW_TASK
3. **包含多种特征**: 按优先级处理：
   - COMMAND > CONTINUE_TASK > NEW_TASK > QUESTION > CHAT

### 高风险操作检测：

检测到高风险关键词（删除、修改、推送、重启等）时：
- 不自动执行
- 返回确认提示
- 用户确认后才执行

---

## 上下文注入

每次执行时注入：
- task objective（任务目标）
- current step（当前步骤）
- completed steps summary（已完成步骤摘要）
- available tools（可用工具）
- relevant memory（相关记忆）
