## 1. 验收范围

### 1.1 核心功能链路
| ID | 功能链路 | 验收标准 |
|----|---------|---------|
| V1 | Telegram E2E 任务创建 | 用户发送 "帮我做X" → 系统创建任务 → 返回确认 |
| V2 | Telegram E2E 任务继续 | 用户发送 "继续" → 系统恢复上下文 → 继续执行 |
| V3 | Telegram E2E 任务状态查询 | 用户发送 "任务状态" → 系统返回当前任务状态 |

### 1.2 作用域隔离
| ID | 隔离项 | 验收标准 |
|----|--------|---------|
| V4 | Chat 隔离 | 不同 chat_id 的任务互不可见 |
| V5 | User 隔离 | 不同 user_id 的任务互不可见 |
| V6 | Scope 绑定 | 任务创建时自动绑定 scope_key |

### 1.3 统一解析
| ID | 解析场景 | 验收标准 |
|----|---------|---------|
| V7 | continue 解析 | 优先级: 显式 task_id > scope active > scope resumable |
| V8 | resume 解析 | 使用 UnifiedTaskResolver 统一逻辑 |
| V9 | run 解析 | 新任务创建使用统一 resolver |

### 1.4 Task Memory 集成
| ID | Memory 类型 | 验收标准 |
|----|------------|---------|
| V10 | Task Memory 保存 | 任务执行后自动保存到 memory |
| V11 | Task Memory 恢复 | 任务恢复时自动加载 memory |

### 1.5 容错与安全
| ID | 安全项 | 验收标准 |
|----|--------|---------|
| V12 | 高风险操作拦截 | delete/push/restart 需二次确认 |
| V13 | 失败路径处理 | 工具执行失败有明确错误信息 |
| V14 | 重启恢复 | 进程重启后可恢复任务状态 |
