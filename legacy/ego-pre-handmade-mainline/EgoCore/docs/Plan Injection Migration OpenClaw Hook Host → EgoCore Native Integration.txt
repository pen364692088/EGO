

# Plan Injection 从 OpenClaw Hook 宿主迁移到 EgoCore 原生集成

# OpenClaw 强制执行版任务单

## 任务名称

**Plan Injection Migration: OpenClaw Hook Host → EgoCore Native Integration**

---

## 一、任务目标

将当前基于 OpenClaw hook 宿主的 Plan Injection 实现，迁移为 **EgoCore 内部原生集成**，确保：

1. Plan Injection 不再依赖 `~/.openclaw/hooks/` 或 OpenClaw gateway hooks
2. EgoCore 自己在回复生成链中直接调用 OpenEmotion `/plan`
3. 注入逻辑只影响回复文案，不影响 task/runtime/control 主链
4. OpenEmotion 不可用时，EgoCore 自动 fallback，主链继续工作
5. OpenClaw 宿主配置、token、hook 安全层不再是此功能前置条件

本轮目标不是扩功能，而是：

## **去掉错误宿主依赖，让 EgoCore ↔ OpenEmotion 集成真正归位。**

---

## 二、当前问题定义

当前错误点：

* Plan Injection 被做成了 OpenClaw hook
* 运行依赖 `~/.openclaw/hooks/` / `~/.openclaw/workspace/hooks/`
* 启动受 OpenClaw gateway `hooks.enabled/hooks.token` 约束
* 这与“OpenEmotion 独立、EgoCore 直接耦合”的当前架构不一致

因此必须纠偏：

## **业务归属、运行承载、部署路径三者都要回到 EgoCore 自己内部。**

---

## 三、硬边界（必须遵守）

### 允许现在做

* 删除/停用 OpenClaw hook 宿主承载
* 将注入逻辑迁入 EgoCore 内部模块
* 直接在 EgoCore reply generation path 中接 `/plan`
* 把配置迁入 EgoCore 配置体系
* 编写迁移后的测试与验收
* 更新文档与运行说明

### 明确禁止

* 不得继续依赖 OpenClaw hooks token / hook loader
* 不得把迁移任务扩成 EgoCore 全量插件系统开发
* 不得引入新的多 Agent 宿主层
* 不得让 `/plan` 影响 task runtime / tool execution / checkpoint / task state / control commands
* 不得顺手做记忆桥、统一人格总线、planner 深耦合
* 不得因为迁移而破坏现有 event mirror / degraded mode

### 核心原则

必须保证：

* EgoCore 是唯一宿主
* OpenEmotion 是独立本地服务
* Plan Injection 是 EgoCore 的内部回复增强能力
* OpenClaw 不再承担此能力的运行时宿主角色

---

## 四、迁移后的目标架构

迁移后必须收敛为：

```text
Telegram
   │
   ▼
EgoCore
   ├─ semantic/task/runtime/control
   ├─ reply generation
   │    ├─ injection gate
   │    ├─ openemotion client (/plan)
   │    ├─ plan adapter
   │    └─ fallback
   └─ event mirror (/event)
            │
            ▼
OpenEmotion
/home/moonlight/Project/Github/MyProject/Emotion/OpenEmotion
```

禁止保留为：

```text
EgoCore → OpenClaw hooks host → OpenEmotion
```

## 五、必须完成的工作

## M1. 停用 OpenClaw hook 宿主承载

### 目标

彻底停止这条错误宿主链继续成为主方案。

### 必须完成

* 停止把 `plan-injection` 作为 OpenClaw hook 的激活路径
* 移除或停用：

  * `~/.openclaw/hooks/` 下相关链接/文件
  * `~/.openclaw/workspace/hooks/` 下相关链接/文件
  * 任何为此功能新增的 `hooks.enabled/hooks.token` 依赖
* 记录迁移前旧路径，便于审计
* 确认停用后 OpenClaw gateway 恢复到和此功能无关的状态

### 验收标准

* Plan Injection 不再依赖 OpenClaw gateway hooks 启动
* 不再需要为此功能配置 `hooks.token`

---

## M2. 在 EgoCore 内新增原生注入模块

### 目标

把计划注入逻辑迁入 EgoCore 内部。

### 建议目录

```text
app/integrations/openemotion/
  client.py
  adapter.py
  fallback.py
  plan_adapter.py
  injection_gate.py
  reply_injection.py
```

### 职责要求

* `client.py`：请求 `/plan`
* `plan_adapter.py`：将 plan 响应映射为回复层结构
* `injection_gate.py`：判断当前消息是否允许注入
* `reply_injection.py`：在回复生成前执行 maybe_inject_plan(...)
* `fallback.py`：统一处理 timeout/refused/5xx/schema invalid/empty plan

### 验收标准

* 注入链代码归属在 EgoCore 内部
* 不再依赖 hook handler.js 作为主实现

---

## M3. 接入 EgoCore 回复生成主链

### 目标

真正进入 EgoCore 原生 reply generation path。

### 必须完成

在 EgoCore 回复生成流程中明确加入：

1. 判定当前是不是允许注入的 reply path
2. 若允许，调用 OpenEmotion `/plan`
3. 成功则只影响回复文案层
4. 失败则普通回复 fallback
5. reply 发出后仍继续原有 `/event` mirror

### 必须跳过的路径

* slash commands
* task control commands
* tool execution path
* waiting/resume control path
* 高风险确认路径

### 验收标准

* normal chat 路径可注入
* command/control/tool path 默认跳过
* 注入不打断原主链

---

## M4. 配置迁入 EgoCore 配置体系

### 目标

让此能力由 EgoCore 自己配置，不再依赖 OpenClaw 宿主配置。

### 必须配置

至少包含：

```yaml
openemotion:
  enabled: true
  base_url: "http://127.0.0.1:18080"
  repo_path: "/home/moonlight/Project/Github/MyProject/Emotion/OpenEmotion"
  auto_start: true
  plan_timeout_ms: 1500
  event_timeout_ms: 800
  inject_plan_into_reply: true
  plan_injection_for_chat_only: true
  skip_plan_for_commands: true
  skip_plan_for_task_control: true
  skip_plan_for_tool_paths: true
  plan_injection_soft_fail: true
```

### 必须完成

* 确认 EgoCore 启动时能读到这些配置
* 确认关闭开关即可恢复原行为
* 配置文档同步更新

### 验收标准

* 配置来源归属 EgoCore
* 不再需要 OpenClaw hooks 配置来启用此能力
