

## EgoCore P2-B 强制执行版任务单（P2-A.2 后修正版）

### 目标

在 **不扩成多 Agent 平台** 的前提下，为 EgoCore 落地 **单 Agent Runtime 的后台推进最小闭环**，并确保后台推进严格遵守 P2-A.2 的真实完成语义，禁止将意图不匹配或后置条件失败的任务误推进为 completed。

---

### 一、硬边界

#### 必须坚持

* 单 Agent Runtime
* Telegram 驱动
* heartbeat 最小续跑
* cron 最小补偿
* 复用现有 runtime 主链
* 复用 UnifiedExecutionResult
* 复用 preflight / tool_doctor
* 复用 IntentMapper / PostconditionValidator 结果

#### 明确禁止

* 多 Agent
* dashboard
* workflow DSL
* 子代理编排
* 新建平行后台执行链
* 绕过 postcondition / intent 验证
* 将 `INTENT_MISMATCH / POSTCONDITION_FAILED / PATH_EXTRACTION_ERROR` 误收口为 completed

---

### 二、必须完成的工作

#### P2-B.1 后台 failure policy

新增后台推进判定层，明确 failure_class → action 映射。

至少覆盖：

* `INTENT_MISMATCH`
* `POSTCONDITION_FAILED`
* `PATH_EXTRACTION_ERROR`
* retryable tool failure
* preflight failure
* safety block
* permission failure
* timeout / transient infra failure

每类必须定义：

* allow_auto_retry
* allow_heartbeat_resume
* allow_cron_resume
* retry_limit
* final_state
* whether_user_notification_required

---

#### P2-B.2 Heartbeat driver

实现 heartbeat 最小驱动器：

* 扫描可推进任务
* 获取 lease / lock
* 调用 `execute_next_step_unified()`
* 写回状态 / checkpoint / event
* 标记 `trigger_source=heartbeat`
* 严格遵守 failure policy
* 不得重试不应自动重试的 failure_class

---

#### P2-B.3 Cron recovery driver

实现 cron 补偿驱动器：

* 周期扫描 durable task store
* 发现 stalled 任务
* 对允许恢复的任务做一次安全补偿推进
* 标记 `trigger_source=cron`
* 处理 missed heartbeat 场景
* 禁止对 `INTENT_MISMATCH / PATH_EXTRACTION_ERROR` 做盲重试

---

#### P2-B.4 Foreground / Background guard

实现前后台隔离保护：

* background 不接管 foreground session
* background 不改写 foreground preference
* background 不向 foreground reply channel 写中间噪声
* session_affinity / task_scope 必须保留

---

#### P2-B.5 Notification policy

实现最小通知闭环：

必须通知：

* completed
* blocked
* manual action required
* intent mismatch blocked
* path extraction blocked
* 用户主动 status 查询

默认不通知：

* heartbeat tick
* cron tick
* retry attempt
* checkpoint save

---

#### P2-B.6 Status query

实现最小状态查询摘要，至少返回：

* task_id
* current_status
* failure_class
* retry_count
* trigger_source
* last_progress_at
* blocked_reason
* manual_action_hint
* last_checkpoint_summary

---

### 三、必须通过的验收

#### 1. 后台主闭环

验证：

* Telegram 发起任务
* 前台执行首轮
* 未完成后进入后台可推进态
* heartbeat 能推进
* heartbeat 缺席时 cron 能补偿
* 最终正确收口

#### 2. 假成功防扩散

验证：

* 错误路径操作不会返回 completed
* `INTENT_MISMATCH` 不会被 heartbeat / cron 继续错误推进为 success
* `POSTCONDITION_FAILED` 不会无限重试
* `PATH_EXTRACTION_ERROR` 会正确 block 或转人工

#### 3. 前后台隔离

验证：

* foreground 不被 background 接管
* foreground preference 不被 background 改写
* background 不制造前台回复串扰

#### 4. 防重入

验证：

* heartbeat 与 cron 不会重复执行同一步
* 同一 task 不会并发双执行
* lease 异常不会造成永久卡死

### 五、强制实施顺序

1. 先做 `failure_class -> background_action` 映射
2. 再做 heartbeat driver
3. 再做 cron recovery
4. 再做 foreground/background guard
5. 再做 notification policy
6. 最后做 status query 和 E2E 验收
