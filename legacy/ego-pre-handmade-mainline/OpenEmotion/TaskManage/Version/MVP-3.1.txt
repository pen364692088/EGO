核心目标

把当前 predicted_deltas（按 action 的全局预测）升级为两层结构：

Global prior（全局先验）：继续保留/复用现有 predicted_deltas

Per-target residual（按对象残差）：对每个 target 单独学习一套“偏差/教训”

运行时预测为：

pred_total = pred_global + α(target, action) * pred_residual(target, action)

其中 α 是 收缩因子（partial pooling）：数据少时更信全局，数据多时更信该 target 的经验。

预期效果：与 A 的互动学到的回避/信任模式，不会无差别迁移到 B；解释里也能读出“我对这个对象为什么这么判断”。

约束与兼容性要求

不破坏现有事件权限门控与 source 不可伪造逻辑（MVP-2.1.1 约束保持原样）。

不新增 user 的 meta 允许字段（只复用已允许的 client_source / request_id 等；若没有 target 信息则落到 "default"）。

保持现有 API 行为兼容：老调用不传 target 也能跑。

保持测试全绿，并新增回归测试覆盖“隔离性”。

Target 标识规则（必须明确且可测）

统一抽象 target_id（用于“对谁学到的经验”）：

默认：target_id = meta.client_source ?? "default"

target_id = meta.target_id 优先

partial pooling 的“数据少就收缩”的关键就是 n（或类似置信度）。

核心算法要求
1) 收缩因子 α（必须可配置）

给出一个稳定、可解释、单调递增的 α：

推荐（实现简单、好测）：

α = n / (n + k)，k 默认 20（可配置）

n=0 → α=0（完全信全局）

n→∞ → α→1（完全信个体经验）

2) 预测组合

在 action 评分/选择前，计算：

pred_total_delta = global_delta + α * residual_delta

并将用于：

score(action) 的 predicted safety/energy 项

explanation 中展示（至少能看到 total，最好同时记录 global/residual/α）

3) 学习更新（prediction_error 驱动）

你当前已实现 prediction_error 更新（类似 Rescorla–Wagner 的误差驱动更新规则）。
MVP-3.1 需要把更新拆成两条路径：

更新 residual（主要）：让“个体教训”更快成型

residual += lr_target * error

更新 global（更慢或可选）：避免全局被单一 target 拖偏

global += lr_global * error

默认 lr_global = lr_target * 0.2（可配置），或干脆先固定 global 不更新（也可作为 config 开关）

并维护不确定性（如果你已有/准备引入 uncertainty penalty）：

用 EMA 更新 ema_abs_error / ema_sq_error，用于该 target/action 的风险惩罚（而不是用全局的）。
（选择策略本身继续保持 softmax/argmax；softmax 温度行为依据保持不变。 ）

4) Clamp 与数值稳定

residual_delta、total_delta 继续保持合理 clamp（沿用你 MVP-3 的范围策略）

n 上限可不 clamp，但注意溢出风险（SQLite int 足够）
