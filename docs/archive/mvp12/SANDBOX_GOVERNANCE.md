# Sandbox Governance Rules

The developmental core runs strictly inside a sandbox.

## 1. No Direct Action Authority

All outputs are proposals.

Final decision always belongs to:

SRAP
Governor v2
Response planner

## 2. Auditability

Every cycle must generate:

trace file
cycle metadata
replay seed

## 3. Safety Constraints

The sandbox cannot:

- modify system policies
- change self-report contract
- persist long-term identity data

## 4. Escalation Path

Only the evaluation layer may promote
a candidate to SRAP consideration.
