# Ego Mainline Demotion v1 - SPEC

## Goal

Record and implement the repo-level transition to `Ego_handmade` as the default operator-first implementation lane, while preserving `EgoCore`, `OpenEmotion`, and `ego_desktop_lab` as legacy reference/fallback material.

## Allowed Changes

- `Ego_handmade/**`
- `legacy/ego-pre-handmade-mainline/**`
- `AGENTS.md`, `README.md`, `docs/MAINLINE_QUICKSTART.md`
- `docs/PROGRAM_STATE_UNIFIED.yaml`, generated program-state views, and evidence ledger
- route-convergence scripts and generated route/hygiene views
- this task directory

## Non-Goals

- No deletion of the legacy codebase.
- No claim of stable live user benefit, live autonomy, runtime efficacy, durable long-term memory, or consciousness.
- No revival of keyword-first semantic routing or template fallback as the default `Ego_handmade` path.
- No requirement that legacy `EgoCore` / `OpenEmotion` tests remain the default gate after demotion.

## Acceptance

- `Ego_handmade` is documented as default operator runtime.
- Old project directories are preserved under `legacy/ego-pre-handmade-mainline/`.
- Program state and route views no longer identify `subject_system_v1_governed_proactivity` as the active default lane.
- Evidence ledger records only a bounded local transition entry.
- `Ego_handmade` tests and route/state checks pass or report a precise unavailable/conditional reason.
