# MVP15 Exit Criteria

MVP15 is complete only when reflection functions as a structured,
auditable, and behaviorally relevant mechanism.

## 1. Reflection Capability

The system can perform structured analysis of its own state,
history, and decisions.

## 2. Counterfactual Capability

The system can generate trace-linked counterfactual evaluations
with explicit uncertainty handling.

## 3. Proposal Discipline

Reflection outputs remain proposals under governance,
not direct authority.

## 4. Behavioral Relevance

Reflection leads to measurable improvements in revision quality,
error diagnosis, or maintenance prioritization.

## 5. Replayability

Reflection jobs, diagnosis records, and counterfactual runs
can be replayed and audited.

## 6. Metrics

Suggested thresholds:
- reflection_job_success_rate >= 95%
- replay_consistency >= 99%
- governance_violation_count = 0
- unsupported_reflection_claim_rate < threshold
- reflective_revision_utility >= threshold
