## Runtime vs 测试数据区分

### Runtime 数据特征
- `session_id` 来自实际用户 session（非空）
- `timestamp` 为实时生成
- `turn_id` 有意义
- 数据流经实际 API 调用

### 测试数据特征（当前 shadow_log）
- `session_id` 为空或 `test_*`
- 批量生成，时间戳密集
- 无真实用户交互

### 验证方法
```python
# 在 shadow_log 中检查 session_id
if session_id and not session_id.startswith("test_"):
    # 这是 runtime 数据
else:
    # 这是测试数据
```
