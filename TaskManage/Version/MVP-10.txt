下面是 **MVP10 · LucidLoop（清明回路）** 的 **cc-godmode 任务拆解版**。目标是：把“像意识”的东西全部压成 **可证伪、可回归、可审计、可对照** 的工程闭环，避免项目跑偏到“只会自述”。

# MVP10 · LucidLoop — cc-godmode Epic

## Epic 目标

实现并验证以下结构的“证据链”：

* Global Workspace（统一焦点 + 广播）
* HOT Self Model（高阶自我表征 + 预测误差校正）
* Valence/Drives 因果调制（风险/探索/计划深度/反思阈值）
* 闭环学习（行动→后果→归因→更新）
* 时间厚度连续性（episodic + narrative + commitments）
* Scientific Mode：干预 + no-report + zombie baseline + evidence battery + Bayes posterior

## Anti-Drift 铁律（防跑偏）

1. **不许用“自我报告”当主要证据**：任何功能都必须落到 no-report 或干预预测上。
2. **每个新增模块必须进入日志因果链**：`module_output -> policy_params -> score -> focus -> plan -> action` 可追溯。
3. **必须有 zombie baseline 对照**：避免“会说 ≠ 会有结构”。
4. **必须可回归**：新增功能必须带单测 + scenario + eval 脚本更新。

### MVP10-T01 — 定义 Schema v1（log/snapshot/plan）

**目标**：固化结构化日志与状态快照，后续模块全部挂靠在此。

  **回滚策略**

### MVP10-T02 — 事件日志写入器 + snapshot 生成

**目标**：每 tick 可写入 JSONL 日志，并可导出 snapshot。
**文件**

* `emotiond/science/ledger.py`（或 `emotiond/logging_mvp10.py`）
  **AC**
* 1 次 run 输出 `artifacts/mvp10/run_<id>.jsonl` + `snapshot_<id>.json`
* 每条 log 带 `tick_id`、`run_id`、`seed`
  **测试**
* `tests/mvp10/test_logging_roundtrip.py`：写入→读回→关键字段一致

### MVP10-T03 — Replay 引擎（mock planner 下确定性）

**目标**：固定 seed + mock planner，replay 能复现 focus/intent/score。
**文件**

* `scripts/replay_mvp10.py`
* `emotiond/loop_mvp10.py`（支持 replay mode）
  **AC**
* replay 后关键字段 hash 一致（或逐项一致）
  **测试**
* `tests/mvp10/test_replay_determinism.py`
  **风险/控制**
* LLM 相关步骤必须可切 mock；任何随机数必须走统一 RNG

### MVP10-T05 — Workspace Arbitrator（竞争选择 + 广播）

**目标**：每 tick 只允许一个 `chosen_focus` 广播到全局。
**文件**

* `emotiond/workspace.py`

* `tests/mvp10/test_workspace_arbitration.py`

### MVP10-T06 — Broadcast 断连干预（disable_broadcast）

**目标**：为科学模式准备“可预测功能分离”。
