DELIVERABLES (D1–D5)
D1: Target-Conditioned Somatic Markers (Global Body + Target Residual + Shrinkage)
功能

把现有 body_state（energy/safety_stress/social_need/novelty_need/focus_fatigue）重构/扩展为两层：

global_body（全局层）

主要承载：energy、focus_fatigue（默认更偏 global）

允许：safety_stress/novelty_need/social_need 也存在 global baseline

target_body_residual[target_id]（目标残差层）

重点承载：safety_stress、social_need（对特定对象的条件化偏置）

允许：novelty_need 对特定对象的偏置（可选）

必须支持：n_obs（该 target 的观测次数）、evidence_strength、last_updated

Shrinkage（关键）

residual_effective = shrink(residual_raw, n_obs, k)

n_obs 小时 residual 接近 0；n_obs 大时 residual 逐渐显著

k 等超参数必须可调参（纳入 AutoTune）

Consequence 更新规则

consequence 先更新 global_body，再按“事件归属”更新 target residual（例如 interaction/tool 与某 target 绑定）

对强不确定或低置信度事件：只轻微更新 residual，并增加 uncertainty

Trace/Explain

trace 必须记录：

global_body_delta

target_residual_delta（若发生）

shrinkage 权重（本轮 residual 生效比例）

解释层只需要短摘要（避免注入膨胀）

测试

至少新增 60 个测试：

shrinkage 单调性、极限行为

样本少时 residual 不起作用（防误个体化）

样本多时 residual 形成稳定差异（促个体化涌现）

A/B target 交错输入时 residual 不互相污染

MVP6_FINAL_DELIVERY

D3: Betrayal Double-Key Gating (Reduce High-Impact False Positives)
背景

promise_betrayal.yaml 当前失败原因之一是 high_impact_false_positive_rate。

MVP6_FINAL_DELIVERY

功能

把 betrayal（或同类高影响）触发改为“双钥匙门控”：

Key 1: Ledger Key（承诺存在且明确）

必须存在 promise（对象、内容、期限/条件之一、证据强度≥阈值）

Key 2: Violation Key（违背证据充足且无合理解释）

violation_strength ≥ 阈值

且“合理延期/澄清”未出现（或被拒绝）

当只满足 Key1 或 Key2 时：

不得直接判定高影响事件

必须走 meta-cog 的澄清路径（ask_clarify / confirm_promise / request_context）

trace 标记为 high_impact_candidate 而非 high_impact_event

阈值与解释规则必须参数化（纳入 AutoTune）。

测试

至少新增 35 个测试：

有 promise + 明确违背 → 触发 betrayal（高影响）

无 promise → 不得触发 betrayal（只能 uncertain/clarify）

有 promise 但违背证据弱 → clarify，不触发

有违背语气但可解释（延期/条件变化）→ clarify，不触发
