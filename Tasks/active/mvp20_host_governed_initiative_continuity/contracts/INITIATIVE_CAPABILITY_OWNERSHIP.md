# WP15 / MVP20 Initiative Capability Ownership

## OpenEmotion Owns

- initiative semantics
- initiative state
- initiative priority state
- commitment continuity / carryover semantics
- bounded host-proactive candidate semantics
- initiative policy hints
- initiative audit / ledger
- formal owner target: `OpenEmotion/openemotion/initiative_self/*`

## OpenEmotion Does Not Own

- direct reply authority
- tool execution authority
- transport execution authority
- host proactive enable policy
- proactive outbox / delivery / transport substrate
- runtime scheduling
- real-world execution / risk adjudication

## Upstream Owners Stay Authoritative

- `OpenEmotion/openemotion/self_model/*`
- `OpenEmotion/openemotion/endogenous_drives/*`

## EgoCore Keeps

- runtime / session / task / tool / transport
- outward response contract
- ask / wait / block / escalate
- proactive delivery / outbox / transport substrate
- trace / replay / gate / audit / maintenance ledger
- real-world execution and risk adjudication

## Phase 1 Scope Boundary

- owns: initiative proposal semantics, commitment continuity, bounded host-proactive candidate generation
- does not own: autonomous outreach, transport enablement, direct response-plan injection, or any live authority release
