# WP7~WP14 Boundary Freeze For WP15

## Frozen Upstream State

- `WP7~WP14` 当前都保持 maintenance / frozen upstream 状态
- 各 phase 的 formal owner、formal read/write path、evidence claim 与 maintenance ledger 仍各自由其 phase authority 管理
- `WP15` 启动不构成任何 upstream reopen

## What WP15 May Not Do

- 不得改写 `WP7~WP14` formal owner contract
- 不得直接回写 `self_model/*`、`endogenous_drives/*`、`reflective_self/*`、`developmental_self/*`、`social_self/*`、`embodied_self/*`、`selfhood_integration/*`
- 不得把 `WP7` host proactive delivery / transport substrate 升格成 `WP15` semantic owner
- 不得把 `WP7~WP14` maintenance institutionalization、controlled evidence 或 frozen upstream status 升格为 `WP15` implementation / mainline / observation / maintenance proof
- 不得因 `WP15` 启动而 reopen `WP7~WP14`
- 不得把 upstream bounded outputs 解释成 live autonomy、direct reply authority、tool authority、或 broader transport maturity
